feature_map

What it does

Telemetry ingestion

Accepts SSH and Apache/Nginx log lines through the API or the local agent, normalizes them, and preserves raw plus structured representations.

Detection stack

Combines brute-force and suspicious-IP heuristics with YAML rules, a rolling-window anomaly detector, and post-alert correlation.

Integrity controls

Hash-chains every persisted log and alert, then exposes verification through a dedicated endpoint.

Operator access

Ships with API-key RBAC, a local tailing agent, Linux install scripts, and systemd units for direct deployment.

system_layout

Architecture overview

agent/api -> normalize -> enrich -> detect -> correlate -> sqlite -> integrity chain -> api -> dashboard

Detection path

Source-aware parsers feed structured events into enrichment, heuristics, YAML rules, anomaly checks, and correlation rules.

Persistence

SQLite stores normalized events, generated alerts, enrichment cache entries, API keys, and chain records for a single-node deployment model.

Operational path

The local agent tails auth and Nginx access logs, tracks offsets across restarts, and forwards batched lines into the same ingestion API.
